# Search Strategy

## Query Patterns

Use multiple narrow searches rather than one broad search:

- `{domain} skill`
- `{domain} "SKILL.md"`
- `{domain} "Claude Code skill"`
- `{domain} "Codex skill"`
- `{domain} "agent skill"`
- `{domain} "awesome"`
- `{domain synonym} workflow`
- non-English keywords when relevant

For writing/admissions domains, include both task terms and artifact names:

- `personal statement`
- `statement of purpose`
- `SOP`
- `admissions essay`
- `motivation letter`
- `study abroad`
- `graduate admissions`
- `留学`
- `文书`

## Evidence Tiers

Tier 1: Directly installable skill

- Has `SKILL.md` or a clear platform skill folder.
- Has clear installation path.
- README and skill description match the user topic.

Tier 2: Skill collection or curated index

- Contains many skill links or subfolders.
- Requires selecting a path before installation.
- Useful as a discovery seed.

Tier 3: Convertible reference

- Not a skill, but has high-quality instructions, workflows, or templates.
- Can be converted into a local skill with attribution and care.

Tier 4: Low-confidence candidate

- Low/no stars, stale, missing docs, unclear ownership, or unrelated SEO content.
- Only include if the domain is sparse and the topic fit is strong.

## Red Flags

- Repository description is keyword-stuffed or unrelated to agent skills.
- No README, no `SKILL.md`, and no clear usage docs.
- Archived repository unless the content is still uniquely valuable.
- Huge copied text dumps with unclear license.
- Star count is high because the repository is a broad list, not because the specific skill is proven.

## Reporting Template

For each candidate, report:

- Name and link
- Stars/forks/last updated
- Type: installable skill, collection, convertible reference, or low-confidence
- Relevant inspected files
- Why it matches
- Risks and caveats
- Suggested install or conversion path
